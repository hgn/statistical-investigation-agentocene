# Example curated repo list for a targeted single-ecosystem analysis, per
# concept.md's cross-repo developer aggregation idea: a deliberately connected
# sample (one ecosystem's core repos) has much higher cross-repo contributor
# overlap than a random sample of unrelated repos, making the pooled
# cross-developer analysis (aisloc.analysis.stats._dev_counterfactual_pooled)
# far more powerful -- the same core maintainers show up across many of these.
#
# Usage:
#   make get-data PROVIDER=list LIST=data/example-rust-core-repos.txt TARGET=40
#
# This is illustrative, not exhaustive or authoritative -- swap in whichever
# ecosystem is of interest (a language, a company's OSS org, a framework's
# plugin ecosystem, ...); the property that matters is "many repos, shared
# core contributors", not "Rust" specifically.
